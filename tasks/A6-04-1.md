# A6-04-1 — Студент должен пройти курс

## Контекст
Публикация шаблонов (draft → pending_review → ready → published) уже работает, но ученик до сих пор не видит курсы на главной. Страница `student-home.ts` содержит TODO и всегда отдаёт пустой список.

## Проблема
- У опубликованного шаблона статус `published`, но ученик всё равно не может его выбрать.
- Страница ученика (`student-home`) не делает запросов к API и остаётся пустой.
- Из UI невозможно создать учебную сессию или продолжить курс.

## Гипотеза решения
1. Добавить API/эндпоинт, который отдаёт опубликованные курсы/уровни ученику (`GET /api/v1/student/courses`).
2. На фронте `student-home.ts` реализовать `loadCourses()` с реальным запросом и отрисовкой карточек.
3. При клике по курсу стартовать сессию (`POST /api/v1/student/sessions`) и вести ученика по заданиям.
4. Учесть role-based доступ: ученики должны видеть только опубликованные темы/уровни.
5. Обновить документацию/инструкцию, что шаг «протестируйте как ученик» делается через новый UI.

## Проверка / критерии принятия
- У ученика на главной появляются карточки курсов (хотя бы один для теста).
- Кнопка «Начать курс» создаёт сессию и переводит на страницу задания.
- Прохождение задания завершает сессию и обновляет статус курса (`in_progress` → `completed`).
- QA может повторить сценарий без ручных запросов к API.
